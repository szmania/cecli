# Development Plan: Integrate Tool Creation and Management

## 1. Summary / Overview
The goal is to integrate tool creation and management capabilities from the `szmania/aider-ce` fork (tag `v0.87.13+fork.1`) into the current codebase. This will empower the agent mode with the ability to dynamically create, load, unload, and manage custom tools, enhancing its problem-solving capabilities. The implementation will adapt the fork's functionality to the existing tool architecture, avoiding the `BaseAiderTool` class and using the specified file paths for tool storage.

## 2. Design Requirements
- **New Commands:** Implement the following user-facing commands:
    - `/tools`: List all available tools.
    - `/tools-create`: Create a new tool from a description.
    - `/tools-load`: Load a tool from a file.
    - `/tools-unload`: Unload a tool.
    - `/tools-move`: Move a tool between local and global scopes.
- **New Core Tools:** Add the following tools to the `aider/tools/` directory:
    - `create_tool.py`
    - Other associated tools from the fork (excluding `base_tool.py`).
- **Tool Storage:**
    - **Local:** `.aider/tools/`
    - **Global:** `~/.aider/tools/`
- **Architecture:**
    - New tools should integrate with the existing tool system.
    - No inheritance from a `BaseAiderTool` class.
    - Prompts must be updated to reflect the new capabilities, especially for the agent.

## 3. Design Specification
- **`aider/tool_manager.py` (New File):**
    - This module will centralize the logic for tool discovery, loading, and management.
    - It will contain functions to:
        - Scan `.aider/tools/` and `~/.aider/tools/` for Python files.
        - Dynamically import and register tools.
        - Handle the creation of new tool files via the `create_tool` tool.
        - Manage loading, unloading, and moving of tools.
- **`aider/commands.py`:**
    - The `Commands` class will be updated to register the new `/tools*` commands.
    - Each new command will call the corresponding function in the `tool_manager`.
- **`aider/coders/agent_coder.py`:**
    - The `AgentCoder` will be modified to use the `tool_manager` to get a list of available tools.
    - It will pass the definitions of these tools to the LLM in the prompt.
- **`aider/coders/agent_prompts.py`:**
    - The system prompt will be updated to include instructions on how to use the `create_tool` function.
    - It will explain that the agent can write new tools for itself to solve complex problems.
- **`aider/tools/create_tool.py` (New File):**
    - This file will contain the implementation of the tool that allows the agent to create other tools. It will take a description and a file name as input and generate the Python code for the new tool.
