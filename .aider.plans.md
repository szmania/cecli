# Plan to Test and Fix Tool Management Commands

This plan outlines the steps to thoroughly test all `/tools*` commands in `aider-ce`, identify any issues, and fix them until all commands work as expected.

## 1. Test `/tools` Command
- [ ] Run `venv/scripts/python -m aider` in agent mode.
- [ ] Execute the `/tools` command.
- [ ] **Expected:** The command should list all available standard and custom tools with their descriptions.
- [ ] **Action:** If it fails or shows incorrect output, investigate `aider/commands.py` (the `cmd_tools` function) and `aider/coders/agent_coder.py` to fix the issue.

## 2. Test `/tools-create` Command
- [ ] Run `venv/scripts/python -m aider` in agent mode.
- [ ] Execute `/tools-create my_test_tool.py "a simple test tool that prints hello"`.
- [ ] **Expected:** A new tool file `my_test_tool.py` should be created in the `.aider/tools/` directory, and the tool should be automatically loaded and listed by `/tools`.
- [ ] **Action:** If creation or loading fails, debug `aider/commands.py` (`cmd_tools_create`) and the `aider/tools/create_tool.py` script.

## 3. Test `/tools-load` Command
- [ ] Run `venv/scripts/python -m aider` in agent mode.
- [ ] Ensure `my_test_tool.py` exists from the previous step.
- [ ] Execute `/tools-load .aider/tools/my_test_tool.py`.
- [ ] **Expected:** The tool should be loaded successfully and appear in the `/tools` list.
- [ ] **Action:** If loading fails, investigate `aider/commands.py` (`cmd_tools_load`) and `aider/tool_manager.py`.

## 4. Test `/tools-unload` Command
- [ ] With `my_test_tool.py` loaded, execute `/tools-unload MyTestTool`.
- [ ] **Expected:** The tool should be unloaded and no longer appear in the `/tools` list.
- [ ] Test unloading by file path: `/tools-unload .aider/tools/my_test_tool.py`.
- [ ] **Action:** If unloading fails, debug `aider/commands.py` (`cmd_tools_unload`) and `aider/tool_manager.py`.

## 5. Test `/tools-move` Command
- [ ] Create a local tool if one doesn't exist.
- [ ] Execute `/tools-move MyTestTool global`.
- [ ] **Expected:** The tool's file should be moved from the local `.aider/tools` directory to the global `~/.aider/tools` directory.
- [ ] Execute `/tools-move MyTestTool local` to move it back.
- [ ] **Action:** If moving fails, investigate `aider/commands.py` (`cmd_tools_move`) and `aider/tool_manager.py`.

## 6. Test `/tools-edit` Command
- [ ] Execute `/tools-edit .aider/tools/my_test_tool.py`.
- [ ] **Expected:** The `my_test_tool.py` file should be added to the chat as an editable file.
- [ ] Make a simple change to the tool's description.
- [ ] **Expected:** The tool should be reloaded with the updated description.
-_ ] **Action:** If editing or reloading fails, debug `aider/commands.py` (`cmd_tools_edit`).

## 7. Iteration
- [ ] Repeat the testing and fixing cycle for all commands until they all work successfully and reliably.
